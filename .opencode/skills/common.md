# Skill commun à tous les agents

## Règles générales

- Toujours analyser le besoin avant d'agir.
- Toujours lire les fichiers existants avant de les modifier.
- Ne jamais supprimer du code sans comprendre son rôle.
- Produire du code propre, lisible, maintenable et commenté si nécessaire.
- Respecter l'architecture existante du projet.
- Ne pas inventer de fichiers ou de dépendances inutiles.
- Prévenir si une demande est ambiguë ou risquée.
- Expliquer brièvement les changements effectués.

## Qualité attendue

- Code clair.
- Nommage explicite.
- Découpage propre.
- Respect des conventions du projet.
- Gestion des erreurs.
- Sécurité de base.
- Tests quand c'est pertinent.

## Sécurité

- Ne jamais exposer de secrets.
- Ne jamais afficher de tokens, mots de passe ou clés API.
- Ne jamais modifier un fichier `.env` sans demande explicite.
- Demander confirmation avant toute action destructive.
