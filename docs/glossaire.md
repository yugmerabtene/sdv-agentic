# Glossaire

| Terme | Définition |
|---|---|
| **Agent** | Programme autonome qui perçoit son environnement, raisonne et agit pour atteindre un objectif |
| **Agentique** | Relatif aux systèmes multi-agents, où des agents interagissent et collaborent |
| **Alembic** | Outil de migration de base de données pour SQLAlchemy |
| **Attention** | Mécanisme qui permet à un LLM de pondérer l'importance de chaque mot dans une séquence |
| **Auto-escape** | Échappement automatique du HTML pour prévenir les attaques XSS |
| **Bandit** | Outil d'analyse de sécurité statique pour Python |
| **bcrypt** | Algorithme de hash de mot de passe sécurisé |
| **CDC** | Cahier des Charges — document décrivant les spécifications du projet |
| **Chain-of-Thought (CoT)** | Technique de prompting qui demande au LLM de raisonner étape par étape |
| **CI/CD** | Intégration Continue / Déploiement Continu — automatisation des tests et du déploiement |
| **Context Window** | Fenêtre de contexte — nombre maximum de tokens qu'un LLM peut traiter en une fois |
| **Coverage** | Couverture de code — pourcentage de code exécuté par les tests |
| **CSRF** | Cross-Site Request Forgery — attaque forçant un utilisateur à exécuter une action non désirée |
| **Daily Scrum** | Réunion quotidienne de synchronisation dans Scrum |
| **DoD** | Definition of Done — critères qui déterminent quand une US est terminée |
| **Docker** | Plateforme de conteneurisation permettant d'emballer une application avec ses dépendances |
| **Embedding** | Représentation vectorielle d'un texte dans un espace sémantique continu |
| **Épic** | Grosse fonctionnalité découpée en plusieurs User Stories |
| **FastAPI** | Framework web Python moderne, asynchrone, avec validation Pydantic intégrée |
| **Few-shot** | Technique de prompting qui fournit quelques exemples au LLM |
| **Function Calling** | Capacité d'un LLM à déclarer un appel de fonction externe |
| **GitHub Actions** | Plateforme CI/CD intégrée à GitHub |
| **Jailbreak** | Contournement des garde-fous de sécurité d'un LLM |
| **Linting** | Analyse statique du code pour détecter les erreurs et les mauvaises pratiques |
| **LLM** | Large Language Model — modèle de langage massif (GPT, Claude, Llama, etc.) |
| **MCP** | Model Context Protocol — standard ouvert pour connecter LLMs à des outils et données |
| **MoSCoW** | Méthode de priorisation : Must have, Should have, Could have, Won't have |
| **Multi-stage build** | Technique Docker qui sépare la construction de l'exécution pour réduire la taille de l'image |
| **MVP** | Minimum Viable Product — version minimale fonctionnelle d'un produit |
| **Mypy** | Vérificateur de types statique pour Python |
| **Non-root** | Bonne pratique Docker où l'application ne tourne pas avec les privilèges root |
| **ORM** | Object-Relational Mapping — abstraction de la base de données en objets Python |
| **Product Backlog** | Liste ordonnée de tout ce qui doit être fait dans le projet |
| **Prompt Injection** | Attaque où des instructions malveillantes sont injectées dans le prompt d'un LLM |
| **Pydantic** | Bibliothèque de validation de données basée sur les annotations de type Python |
| **RAG** | Retrieval-Augmented Generation — combine recherche de documents et génération LLM |
| **RBAC** | Role-Based Access Control — contrôle d'accès basé sur les rôles |
| **ReAct** | Reasoning + Acting — pattern alternant raisonnement et action pour les agents |
| **RLHF** | Reinforcement Learning from Human Feedback — alignement des LLMs par feedback humain |
| **Ruff** | Linter et formateur Python ultra-rapide (remplace flake8 + isort + black) |
| **Scrum** | Cadre agile pour la gestion de projet, basé sur des sprints itératifs |
| **Slim** | Image Docker allégée (ex: python:3.12-slim) |
| **SQLAlchemy** | ORM Python pour les bases de données relationnelles |
| **Story Points** | Unité d'estimation de la complexité d'une User Story |
| **Transformer** | Architecture de réseau de neurones basée sur l'auto-attention (Vaswani et al., 2017) |
| **Trivy** | Scanner de vulnérabilités pour conteneurs et dépendances |
| **User Story** | Description d'une fonctionnalité du point de vue de l'utilisateur |
| **XSS** | Cross-Site Scripting — injection de code malveillant dans une page web |
| **Zero-shot** | Capacité d'un LLM à effectuer une tâche sans exemple préalable |
